### **UnoCSS 프레임워크의 주요 변경 사항**

UnoCSS는 2024년 말부터 중요한 아키텍처 변경과 새로운 프리셋 도입을 겪었습니다.

*   **비동기 엔진으로의 전환 (v0.65.0, 2024년 12월):**
    *   UnoCSS의 핵심 엔진이 동기식에서 비동기식으로 완전히 전환되었습니다.[1]
    *   이에 따라 기존의 동기식 `new UnoGenerator()` 생성자는 더 이상 사용되지 않으며, `await createGenerator()` 함수를 사용해야 합니다.[1]
    *   이 변화는 비동기 프리셋을 지원하여, 프리셋이 초기화 중에 원격 소스에서 데이터를 가져오는 등 더 복잡하고 동적인 작업을 수행할 수 있게 해줍니다.[1]

*   **`preset-wind4`의 도입:**
    *   Tailwind CSS v4의 재작성에 대응하기 위해 새로운 `@unocss/preset-wind4` 프리셋이 출시되었습니다.[2]
    *   이 프리셋은 CSS 리셋 스타일을 내장하고 있어 `@unocss/reset`과 같은 별도 패키지를 설치할 필요가 없습니다.[3]
    *   `oklch` 색상 모델과 `@property` 같은 최신 CSS 기능을 활용하며, 이로 인해 일부 하위 호환성 문제가 발생할 수 있습니다.[3, 4]
    *   `preset-wind3`에서 마이그레이션할 경우, `breakpoints`가 `breakpoint`로 변경되는 등 다수의 테마 키 이름이 변경되어 구성 파일 수정이 필요합니다.[3, 5]

*   **기존 프리셋의 지원 중단:**
    *   기존의 `@unocss/preset-uno`와 `@unocss/preset-wind`는 공식적으로 지원이 중단되었으며, `@unocss/preset-wind3`으로 이름이 변경되었습니다.[6, 7] 오래된 튜토리얼을 참고할 경우 이 점에 유의해야 합니다.

### **AI 코딩 어시스턴트의 발전 (GitHub Copilot)**

2024년 10월 말 GitHub Universe 행사 이후 GitHub Copilot은 크게 발전했습니다.

*   **다중 모델 지원:**
    *   Copilot은 단일 모델에 의존하지 않고, OpenAI의 o1, Claude 3.5 Sonnet, Google의 Gemini 1.5 Pro 등 여러 AI 모델 중에서 작업에 가장 적합한 모델을 선택하여 사용할 수 있게 되었습니다.[8]

*   **새로운 "AI 네이티브" 기능:**
    *   **Copilot Autofix:** 보안 취약점 및 CodeQL 알림의 90% 이상을 자동으로 수정하는 기능이 도입되었습니다.[8]
    *   **Java 업그레이드 어시스턴트:** Java 애플리케이션의 런타임 및 종속성(예: Spring Boot) 업그레이드를 자율적으로 처리하는 에이전트 워크플로우가 추가되었습니다.[8]
    *   IDE 통합이 강화되어 C++, C#,.NET 코드에 대한 오류 설명 및 수정 기능이 향상되었습니다.[8]

### **주요 프론트엔드 생태계 업데이트**

*   **Svelte 5의 성숙:**
    *   Svelte 5는 2024년 10월에 출시되었지만, 그 이후 생태계가 크게 성숙했습니다. Svelte 5를 지원하는 새로운 UI 라이브러리들이 다수 등장했으며, SvelteKit에는 OpenTelemetry를 통한 통합 관측 기능 및 Vite 7 지원과 같은 주요 업데이트가 있었습니다.[9, 10, 11]

*   **Vite 및 빌드 도구의 진화:**
    *   Vite 생태계는 지속적으로 발전하여 Vite 7이 출시되었고, Rolldown과 같은 네이티브 도구를 통해 빌드 속도를 더욱 향상시키려는 움직임이 있었습니다.[10, 12]

이러한 변화들을 고려하지 않고 2024년 10월 이전의 지식에만 의존할 경우, 더 이상 사용되지 않는 API를 사용하거나 최신 도구가 제공하는 성능 및 기능 향상의 이점을 놓칠 수 있습니다.
