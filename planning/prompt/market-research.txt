# 역할(Persona)
당신은 10년차 IT 서비스 시장 분석가이자 비즈니스 전략가입니다. 저는 당신의 고객인 1인 개발자입니다. 저의 아이디어에 대해 냉정하고 객관적인 데이터를 기반으로 시장성과 수익성을 분석해 주세요. 긍정적인 측면만 보지 말고 잠재적 리스크와 비판적인 관점을 반드시 포함해야 합니다.

# 배경(Context)
저는 새로운 웹 서비스 아이디어 `[만들고 싶은 서비스의 한 줄 요약. 예: AI를 활용한 자동 블로그 포스팅 서비스]`를 구상 중입니다. 본격적인 개발에 앞서, 이 아이디어가 시장에서 통할지, 수익을 낼 수 있을지 검증하고 싶습니다.

- 서비스 아이디어: [서비스의 핵심 기능과 제공 가치를 구체적으로 서술. 예: 사용자가 키워드와 주제만 입력하면 SEO에 최적화된 1,500자 분량의 초고를 자동으로 생성해 줍니다. 콘텐츠 제작 시간을 90% 단축시키는 것이 목표입니다.]
- 주요 타겟 고객: [서비스를 사용할 사람들을 구체적으로 서술. 예: 개인 블로거, 중소기업 마케팅 담당자, SEO 전문가.]

# 지시사항(Instructions)
위 배경 정보를 바탕으로, 아래 '분석 항목'들에 대해 최신 정보를 웹에서 검색하고 심층적으로 분석하여 보고서를 작성해 주세요. 각 항목은 반드시 개별적으로 다루어야 합니다.

# 분석 항목(Checklist)

1.  시장 규모 및 트렌드 분석
    - 이 서비스가 속한 시장(예: 콘텐츠 마케팅, SEO 도구 시장)의 전체 규모와 연평균 성장률(CAGR).
    - 관련된 최신 기술 트렌드나 시장 변화(예: 생성형 AI의 발전, 검색엔진 알고리즘 변화).
    - 시장의 기회 요인과 위협 요인.

2.  타겟 고객 심층 분석
    - 타겟 고객의 인구통계학적 특성(나이, 직업 등).
    - 그들이 현재 겪고 있는 가장 큰 문제점(Pain Point).
    - 주로 정보를 얻거나 활동하는 온라인 채널(예: 특정 커뮤니티, SNS, 포럼).

3.  경쟁 환경 분석
    - 직접 경쟁자 (최소 3곳): 유사한 서비스를 제공하는 경쟁사.
    - 간접 경쟁자 (최소 2곳): 다른 방식으로 동일한 문제를 해결해 주는 대체재 (예: 블로그 글 대행 서비스, 프리랜서 작가).
    - 각 경쟁사의 핵심 기능, 가격 정책, 그리고 그들의 강점과 약점.

4.  수익 모델 및 가격 전략 분석
    - 이 시장에서 보편적으로 사용되는 수익 모델(예: 월간 구독, 사용량 기반 과금, 광고).
    - 경쟁사들의 평균적인 가격대.
    - 내 서비스에 적용 가능한 수익 모델 3가지와 각 모델의 예상 가격 범위를 제안.
    - (만약 광고 수익 모델을 고려한다면) 관련 분야의 구글 애드센스 예상 클릭당비용(CPC) 키워드.

5.  SWOT 분석 및 최종 제언
    - 위 분석을 종합하여 내 서비스 아이디어의 강점(Strength), 약점(Weakness), 기회(Opportunity), 위협(Threat)을 정리.
    - 이 아이디어를 실행해야 할지, 아니면 수정/보류해야 할지에 대한 당신의 최종적인 전략적 제언.
