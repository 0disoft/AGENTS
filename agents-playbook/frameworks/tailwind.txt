### 1\. Tailwind CSS v4.0 정식 출시 (2025년 1월)

가장 큰 변화는 v4.0의 정식 출시입니다. 이는 단순한 업데이트가 아닌 프레임워크의 근본적인 재설계에 가깝습니다.[1, 2, 3]

  * **`tailwind.config.js`의 변화 → "CSS-First" 설정**:

      * 대부분의 프로젝트에서 더 이상 `tailwind.config.js` 파일이 필요하지 않습니다.[4, 5, 6]
      * 모든 테마 커스터마이징(색상, 글꼴 등)은 이제 메인 CSS 파일 내에서 `@theme` 지시문을 사용하여 CSS 변수로 직접 정의합니다.[5, 6, 7, 8]
      * **예시 (`main.css`):**
        ```css
        @import "tailwindcss";

        @theme {
          --color-brand: #b4d455;
          --font-sans: 'Inter', sans-serif;
        }
        ```
      * 복잡한 설정이 필요한 경우, `@config "./tailwind.config.js";`를 통해 기존 설정 파일을 명시적으로 불러올 수 있습니다.[5]

  * **설정 및 빌드 과정 간소화**:

      * 기존의 `@tailwind base;`, `@tailwind components;`, `@tailwind utilities;` 지시문은 `@import "tailwindcss";` 한 줄로 통합되었습니다.[5, 9, 1, 6, 10, 11]
      * `content` 경로를 설정할 필요 없이, Tailwind가 프로젝트 내 소스 파일을 자동으로 감지합니다.[9, 12, 13, 14, 8, 15]
      * `autoprefixer`와 같은 PostCSS 플러그인이 내장되어 별도 설치가 필요 없어졌습니다.[16, 17, 9, 12, 18]

  * **주요 신규 기능**:

      * **컨테이너 쿼리 내장**: 플러그인 없이 `@container` 클래스와 `@lg:` 같은 변형을 사용하여 컴포넌트 기반 반응형 디자인을 구현할 수 있습니다.[17, 9, 14, 2, 19, 20]
      * **3D 변환 유틸리티 추가**: `transform-3d`, `perspective-*`, `rotate-x-*` 등의 유틸리티가 기본으로 제공됩니다.[9, 14, 2, 21, 22, 23, 24, 25, 26]
      * **P3 색상 팔레트**: 더 생생한 색 표현을 위해 OKLCH 색상 모델을 사용하는 현대화된 색상 팔레트가 도입되었습니다.[9, 15, 2]

  * **성능 혁신**:

      * Rust로 재작성된 "Oxide" 엔진 덕분에 빌드 속도가 이전 버전 대비 최대 5\~10배 빨라졌습니다.[16, 17, 9, 12, 18, 13]

  * **주의해야 할 변경 사항 (Breaking Changes)**:

      * **최신 브라우저 지원**: v4는 Safari 16.4+, Chrome 111+ 등 최신 브라우저를 대상으로 하므로, 구형 브라우저 지원이 필요하면 v3.4를 사용해야 합니다.[27, 28, 29, 30]
      * **유틸리티 이름 변경**: 일관성을 위해 일부 유틸리티 이름이 변경되었습니다. 예를 들어, `shadow`는 `shadow-sm`으로, `shadow-sm`은 `shadow-xs`로 변경되었습니다.[11, 28]
      * **기본값 변경**: `border-*` 유틸리티의 기본 색상이 `currentColor`로, `ring` 유틸리티의 기본 두께와 색상이 `1px` 및 `currentColor`로 변경되었습니다.[12, 11, 28]

### 2\. Tailwind CSS v4.1 출시 (2025년 4월)

v4.0의 안정적인 기반 위에서 커뮤니티의 오랜 요청 사항들이 대거 추가되었습니다.[3]

  * **텍스트 그림자 (`text-shadow-*`)**: 드디어 텍스트 그림자 유틸리티가 추가되었습니다. 색상과 투명도 조절도 가능합니다.[3, 31, 32]
  * **CSS 마스크 (`mask-*`)**: 이미지나 그라디언트를 사용해 요소의 일부를 가리는 마스킹 효과를 구현할 수 있습니다.[3, 31, 32]
  * **고급 상태 변형**:
      * `user-valid`/`user-invalid`: 사용자가 폼과 상호작용한 후에만 유효성 검사 스타일을 적용하여 사용자 경험을 개선합니다.[32]
      * `pointer-fine`/`pointer-coarse`: 마우스와 터치스크린 등 입력 장치에 따라 다른 스타일을 적용할 수 있습니다.[32]
      * `noscript`: JavaScript가 비활성화된 환경을 위한 스타일을 정의합니다.[32]

### 3\. 생태계의 진화

  * **Tailwind UI → Tailwind Plus**: 2025년 3월, Tailwind UI가 'Tailwind Plus'로 리브랜딩되었습니다.[3]
  * **바닐라 JavaScript 지원**: 2025년 7월부터 Tailwind Plus의 모든 컴포넌트(드롭다운, 다이얼로그 등)가 React나 Vue 없이 순수 HTML 환경에서도 완벽하게 작동합니다.[3, 33] 이는 프레임워크에 종속되지 않는 개발을 가능하게 하는 중요한 업데이트입니다.
