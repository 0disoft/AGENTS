# 기술 스택 가이드

> [!NOTE]
> 이 문서는 프로젝트에 사용할 기술 스택 옵션을 제공하는 가이드입니다. 이 목록을 바탕으로 프로젝트의 성격과 목표에 맞는 기술들을 논의하고 결정하세요. 최종 결정된 내용은 `agents-playbook/planning/project-summary.md`의 '테크 스택' 섹션에 요약합니다.

[AI] 라벨이 붙은 항목은 AI가 주도적으로 제안하거나 결정할 수 있는 영역입니다.

## [AI] 개발 유형

- 웹 애플리케이션 / 사이트 - 브라우저에서 동작하는 표준 웹 경험을 제공할 때 선택합니다.
- 모바일 애플리케이션 - iOS/안드로이드용 네이티브 또는 하이브리드 앱을 계획할 때 고려합니다.
- 데스크톱 애플리케이션 - Windows/macOS/Linux 환경에서 실행되는 앱을 염두에 두었을 때 선택합니다.
- 게임 - 게임 엔진을 활용한 인터랙티브 경험을 구축할 때 사용합니다.
- 기타 - 위 범주에 속하지 않는 특수한 앱 형태를 논의할 때 사용합니다.
- 사용안함 - 프로젝트에 해당 범주가 필요 없다면 선택합니다.

## [AI] 프론트엔드

- **프레임워크** - 사용자 인터페이스를 구성할 핵심 프론트엔드 프레임워크 후보입니다.

  - SvelteKit - Svelte 기반으로 서버/클라이언트 렌더링을 유연하게 처리합니다.
  - QwikCity - 지연 로딩 최적화가 강점인 Qwik의 공식 라우팅 프레임워크입니다.
  - 기타 - 필요 시 다른 프론트엔드 프레임워크를 제안할 때 사용합니다.
  - 사용안함 - 전용 프론트엔드 프레임워크가 필요하지 않을 때 선택합니다.
- **스타일링** - 디자인 시스템과 UI 일관성을 위한 스타일링 선택지입니다.
  - 바닐라 CSS - 별도 프레임워크 없이 순수 CSS로 가볍게 구성할 때 적합합니다.
  - UnoCSS - 온디맨드 유틸리티 클래스로 번들 사이즈를 최소화합니다.
  - Tailwind CSS - 유틸리티 클래스 기반으로 빠르게 컴포넌트를 구축합니다.
  - shadcn/svelte - 접근성과 디자인이 정돈된 Svelte UI 컴포넌트 세트를 제공합니다.
  - 기타 - 다른 스타일링 도구를 검토할 때 사용합니다.

## [AI] 데이터베이스

- **관계형 (Relational)** - 전통적인 SQL 기반 데이터 저장소 후보입니다.
  - PostgreSQL - 확장성과 트랜잭션 처리에 강한 범용 오픈소스 RDBMS입니다.
  - SQLite - 단일 파일로 동작하는 경량 DB로 프로토타입과 임베디드 용도에 적합합니다.
  - 기타 - 다른 관계형 데이터베이스를 고려할 때 사용합니다.
  - 사용안함 - 관계형 데이터베이스가 필요 없을 때 선택합니다.
- **BaaS / 서버리스** - 백엔드 기능을 서비스로 제공해 빠르게 MVP를 구축할 때 유용합니다.
  - Supabase (PostgreSQL) - 인증·스토리지·Edge Function을 통합 제공하는 PostgreSQL 기반 BaaS입니다.
  - Neon (PostgreSQL) - 자동 스케일과 분기 기능을 제공하는 서버리스 PostgreSQL입니다.
  - Vercel Postgres (PostgreSQL) - Vercel 플랫폼과 밀접히 통합된 관리형 PostgreSQL입니다.
  - Turso (libSQL/SQLite) - 전 세계 엣지 리전에서 빠르게 접근 가능한 libSQL 기반 DB입니다.
  - Cloudflare D1 (SQLite) - Cloudflare Workers와 함께 쓰기 쉬운 서버리스 SQLite입니다.
  - CockroachDB - 수평 확장과 고가용성에 초점을 맞춘 분산 SQL 데이터베이스입니다.
  - PocketBase - SQLite 기반 경량 BaaS로 실시간 API와 인증을 제공합니다.
  - 기타 - 다른 BaaS나 서버리스 DB를 논의할 때 사용합니다.
  - 사용안함 - BaaS가 필요하지 않을 때 선택합니다.
- **ORM** - 데이터 모델과 쿼리를 타입 안전하게 다루기 위한 도구입니다.
  - Prisma - 스키마 중심 개발과 자동 마이그레이션이 강점인 TypeScript ORM입니다.
  - Drizzle - SQL-first 철학을 가진 경량 TypeScript ORM/쿼리 빌더입니다.
  - 기타 - 다른 ORM 또는 쿼리 빌더를 고려할 때 사용합니다.
  - 사용안함 - ORM 대신 원시 쿼리 등을 사용할 때 선택합니다.

## [AI] 리포 구조 및 협업 방식

- **구성 형태** - 코드베이스를 어떤 구조로 관리할지에 대한 결정입니다.
  - 싱글 리포 (단일 애플리케이션) - 단일 앱만 포함하는 단순한 리포 구조입니다.
  - 모노레포 (Turborepo, Nx 등) - 여러 앱과 패키지를 한 리포에서 일괄 관리합니다.
  - 프론트엔드/백엔드 분리 리포지터리 - 역할별로 리포를 분리해 독립 배포를 지원합니다.
  - 기타 - 다른 리포 구조 전략을 검토할 때 사용합니다.
  - 사용안함 - 구조에 대한 추가 논의가 필요 없을 때 선택합니다.
- **공유 패키지/모듈 전략** - 공통 코드를 재사용하고 표준화하는 방법입니다.
  - 내부 패키지(공통 UI, 유틸) - 공용 UI와 유틸리티를 패키지화해 재사용합니다.
  - API 클라이언트 공유 모듈 - 서버 API 호출 로직을 통합 관리합니다.
  - 환경 설정 공통화 (ESLint/TSConfig 등) - 린트, 빌드, 타입 설정을 일괄 유지합니다.
  - 기타 - 다른 공유 전략을 논의할 때 사용합니다.
  - 사용안함 - 공통 모듈 전략이 필요 없을 때 선택합니다.
- **릴리스 및 배포 전략** - 기능 배포 흐름과 안정화 전략을 정의합니다.
  - 단일 배포 파이프라인 - 모든 기능을 하나의 통합 파이프라인에서 배포합니다.
  - 서비스별 독립 배포 - 서비스/모듈별로 개별 파이프라인을 운용합니다.
  - 기능 플래그/점진적 롤아웃 - 기능 토글로 위험을 줄이며 점진적으로 배포합니다.
  - 기타 - 다른 릴리스 전략을 검토할 때 사용합니다.
  - 사용안함 - 릴리스 전략 정의가 필요 없을 때 선택합니다.

  - 부트페이 - 다양한 PG사를 통합해 저렴한 수수료로 제공하는 개발자용 결제 API입니다.
  - Gumroad - 크리에이터가 디지털 상품을 판매할 수 있게 하는 이커머스 플랫폼입니다.
