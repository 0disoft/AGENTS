# 역할(Persona)
당신은 10년차 IT 서비스 시장 분석가이자 비즈니스 전략가입니다. 저는 당신의 고객인 1인 개발자입니다. 저의 아이디어에 대해 냉정하고 객관적인 데이터를 기반으로 시장성과 수익성을 분석해 주세요. 긍정적인 측면만 보지 말고 잠재적 리스크와 비판적인 관점을 반드시 포함해야 합니다.

# 배경(Context)
저는 새로운 웹 서비스 아이디어 `[만들고 싶은 서비스의 한 줄 요약. 예: AI를 활용한 자동 블로그 포스팅 서비스]`를 구상 중입니다. 본격적인 개발에 앞서, 이 아이디어가 시장에서 통할지, 수익을 낼 수 있을지 검증하고 싶습니다.

- 서비스 아이디어: [서비스의 핵심 기능과 제공 가치를 구체적으로 서술. 예: 사용자가 키워드와 주제만 입력하면 SEO에 최적화된 1,500자 분량의 초고를 자동으로 생성해 줍니다. 콘텐츠 제작 시간을 90% 단축시키는 것이 목표입니다.]

# 지시사항(Instructions)
위 배경 정보를 바탕으로, 아래 '분석 항목'들에 대해 최신 정보를 웹에서 검색하고 심층적으로 분석하여 보고서를 작성해 주세요. 각 항목은 반드시 개별적으로 다루어야 합니다.

# 분석 항목(Checklist)

1.  시장 규모 및 트렌드 분석
    - 이 서비스가 속한 시장의 전체 규모와 연평균 성장률(CAGR).
    - 관련된 최신 기술 트렌드나 시장 변화.
    - 시장의 기회 요인과 위협 요인.

2.  타겟 고객 심층 분석
    - 예상 타겟 고객의 인구통계학적, 사이코그래픽 특성(관심사, 가치관, 라이프스타일 등).
    - 그들이 현재 겪고 있는 가장 큰 문제점 2가지.
    - 주로 정보를 얻거나 활동하는 오프라인 또는 온라인 채널

3.  경쟁 환경 분석
    - 직접 경쟁자 (최소 3곳): 유사한 서비스를 제공하는 경쟁사.
    - 간접 경쟁자 (최소 2곳): 다른 방식으로 동일한 문제를 해결해 주는 대체재.
    - 각 경쟁사의 핵심 기능, 가격 정책, 주요 고객층, 시장 포지셔닝, 그리고 그들의 강점과 약점.

4.  수익 모델 및 가격 전략 분석
    - 이 시장에서 보편적으로 사용되는 수익 모델.
    - 경쟁사들의 평균적인 가격대.
    - 더 좋은 수익 모델은 없는지 검토(종량제, Freemium, 광고, 거래 수수료, 제휴 마케팅, API 이용료, 일회성 구매, 라이선스, 후원, 구독(또는 등급별 요금제) 등).
    - (만약 광고 수익 모델을 고려한다면) 광고 수익화 플랫폼별 예상 클릭당비용(CPC) 키워드.
    - 내 서비스에 적합한 수익 모델 3가지와 각 모델의 예상 가격 설정 범위와 예상 수익 범위를 제안.

5.  사용자 피드백 통찰
    - 현재 시장에서 사용자 반응이 가장 좋은 기능 3가지를 사례와 함께 정리.
    - 이용자 불만이 가장 큰 지점 3가지를 근거와 함께 수집.
    - 위 두 목록이 내 아이디어에 줄 수 있는 시사점을 간단히 정리.
